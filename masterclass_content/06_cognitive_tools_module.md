# 模块 6: 认知工具 - 工程化 AI 的思维过程

### 模块概要

本模块标志着我们方法的重大转变。我们超越提供上下文,转向积极工程化 AI 的推理过程本身。"认知工具"是模仿人类心理捷径(启发式)的高级结构化方法,使 AI 的思维更加强大、可重用和透明。

我们将探索三个关键工具。首先,**提示程序**(或协议壳),它将我们的提示从一次性命令转变为可重用的、可编程的模板。其次,**上下文模式**,它结构化我们提供给 AI 的信息,就像数据库模式组织数据一样。最后,**递归提示**,一种强大的技术,创建一个反馈循环,迫使 AI 反思并改进自己的工作。通过掌握这些工具,您从提示撰写者进化为真正的"认知架构师",设计 AI 用于思考的系统本身。

### 关键要点

*   **编程您的提示:** "提示程序"是一个具有定义参数、清晰过程和指定输出格式的可重用提示模板。它将提示从艺术转变为工程学科。
*   **结构不是可选的:** "上下文模式"(例如,使用 JSON 或 YAML)定义上下文的形状,使其可预测并更容易让 AI 理解。这减少了歧义并提高了可靠性。
*   **强制自我纠正:** "递归提示"是一个循环,您要求 AI 批评自己先前的答案。这个简单的反馈机制可以显著提高复杂输出的质量和准确性。
*   **用于清晰度的协议壳:** "协议壳"是一种简单的、基于文本的方式来实现提示程序,无需代码,使用清晰的 `/protocol.name{}` 语法来定义意图、输入、过程和输出。
*   **成为认知架构师:** 这些工具的目标是将您的工作从编写单个提示提升到为 AI 使用设计可扩展、可重用和透明的推理系统。

> **专业提示:** 回到您最复杂和最成功的提示——您最自豪的那个。分析它并尝试将其逆向工程为"提示程序"。形式化核心 `任务`,识别您引导 AI 经历的隐式 `过程` 步骤,并定义您期望的 `输出格式`。这种解构您自己直觉成功的行为是使您的提示技能更加系统化和可扩展的巨大一步。

### 轮到您了:迷你挑战

您的目标是将"提示程序"概念应用于标准的即席提示,使其更加结构化和可重用。

**任务:**
您有以下简单的"原子"提示:
`"向一个 10 岁的孩子用两段话解释光合作用的概念。"`

**您的挑战:**
使用下面的模板将这个即席提示重写为结构化的**提示程序**。您的程序应该有 `概念`、`目标受众` 和 `长度` 的清晰参数。

**要填写的模板:**
```
program ExplainConcept(concept, target_audience, length) {
  // 定义任务
  task = `...`;

  // 定义过程
  process = ```
    1. ...
    2. ...
    3. ...
  ```;

  // 定义输出格式
  format = `...`;

  // 构建完整的提示
  return `${task}\n\n过程:\n${process}\n\n格式:\n${format}\n\n要解释的概念: ${concept}`;
}
```
这个练习将为您提供实践经验,将提示视为可重用的结构化组件,而不是单个命令。
