## 建议的翻译方法

### 方法 1: 分段翻译
将大文件分成多个逻辑部分,逐部分翻译:

```bash
# 示例: 翻译 recursive.emergence.shell.md
# 第 1 部分: 1-400 行 (引言和概念)
# 第 2 部分: 401-800 行 (协议定义)
# 第 3 部分: 801-1265 行 (实现和应用)
```

### 方法 2: 优先级翻译
按使用频率和重要性排序翻译:

1. 高优先级:
   - `attractor.co.emerge.digest.md` (摘要 - 快速参考)
   - `recursive.emergence.shell.md` (核心协议)
   - `memory.reconstruction.attractor.shell.md` (记忆系统)

2. 中优先级:
   - `recursive.memory.attractor.shell.md`
   - `field.resonance.scaffold.shell.md`

3. 低优先级:
   - `context.memory.persistence.attractor.shell.md`
   - `field.self_repair.shell.md`

### 方法 3: 摘要优先
先翻译所有摘要文件(较短),然后再处理完整 shell 文档:

1. 翻译 `attractor.co.emerge.digest.md` (~180 行)
2. 创建其他协议的摘要(如果不存在)
3. 然后翻译完整 shell 文档
