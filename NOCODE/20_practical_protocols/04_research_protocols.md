# 研究协议

> *"研究是正式化的好奇心。它是有目的的探索和调查。"*
>
> **— Zora Neale Hurston**

## 研究协议简介

研究协议将通常混乱、非线性的知识发现过程转化为结构化、高效的工作流程,始终如一地产生可靠的洞察。通过为调查和分析建立明确的框架,这些协议帮助您清晰而有目的地导航复杂的信息环境。

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│            研究协议的优势                            │
│                                                     │
│  • 系统化的知识发现和验证                            │
│  • 减少分析中的认知偏差                              │
│  • 高效探索复杂主题                                  │
│  • 从问题到洞察的清晰进程                            │
│  • 可追溯的推理和证据路径                            │
│  • 可重复的知识开发过程                              │
│                                                     │
└─────────────────────────────────────────────────────┘
```

本指南提供即用型研究协议,适用于常见的知识探寻场景,并配有实施指导和性能指标。每个协议遵循我们的NOCODE原则:导航、编排、控制、优化、部署和演化。

## 如何使用本指南

1. **选择协议**,与您的研究目标匹配
2. **复制协议模板**,包括提示词并自定义
3. **在交互开始时**向AI助手提供完整协议
4. **遵循结构化流程**,从初始问题到验证洞察
5. **监控指标**以评估有效性
6. **迭代和优化**协议以供未来研究使用

**苏格拉底式问题**:您目前发现哪些类型的研究最具挑战性或最耗时?您通常在知识发现过程的哪些环节遇到瓶颈?

## 1. 文献综述协议

**何时使用此协议:**
需要全面了解某个主题的现有知识?此协议引导您系统性地探索可用信息——非常适合主题概述、最新技术评估或知识综合。

```
提示词: 我需要对量子机器学习算法的最新进展进行全面的文献综述。我特别关注过去三年的发展、该领域新兴的实际应用,以及仍需克服的最重大技术挑战。请帮助我系统性地概述该领域的知识现状。

协议:
/research.literature{
    intent="全面理解某主题的现有知识",
    input={
        research_topic="量子机器学习算法的最新进展(过去三年)",
        key_questions=[
            "量子机器学习算法最重要的理论进展是什么?",
            "这些进展产生了哪些实际应用?",
            "该领域仍需克服哪些技术挑战?"
        ],
        scope_boundaries={
            timeframe: "过去三年(2022-2025)",
            inclusion: "同行评审研究、主要实验室的技术报告、重要预印本",
            exclusion: "科普报道、入门材料、2022年之前的基础研究"
        },
        organization_approach="主题分析,主题内按时间顺序推进"
    },
    process=[
        /map{
            action="创建该领域的概念框架",
            elements=[
                "关键理论基础",
                "主要研究流派",
                "重要应用",
                "演化中的术语和概念"
            ]
        },
        /explore{
            action="识别和分析关键贡献",
            for_each="research_stream",
            elements=[
                "开创性工作和突破",
                "方法论创新",
                "实证发现和结果",
                "局限性和争议"
            ]
        },
        /synthesize{
            action="发展整合性理解",
            approaches=[
                "识别新兴模式和趋势",
                "映射研究流派之间的关系",
                "对比竞争性理论或方法",
                "注意共识观点和开放性问题"
            ]
        },
        /evaluate{
            action="评估研究的质量和重要性",
            criteria=[
                "方法论严谨性",
                "实证支持",
                "理论一致性",
                "实际意义"
            ]
        },
        /organize{
            action="将发现结构化为连贯框架",
            elements=[
                "主题组织",
                "主题内按时间顺序推进",
                "突出关系和对比",
                "识别差距和机会"
            ]
        }
    ],
    output={
        knowledge_synthesis="知识现状的全面概述",
        key_findings="最重要的洞察和发展的摘要",
        research_map="该领域的可视化或结构化表示",
        gap_analysis="识别未解答的问题和机会"
    }
}
```

### 实施指南

1. **主题定义**:
   - 定义具体的焦点和范围
   - 制定指导探索的关键问题
   - 考虑广度和深度要求

2. **范围边界设置**:
   - 建立清晰的时间框架参数
   - 定义纳入和排除标准
   - 考虑资源限制和优先级

3. **组织方法选择**:
   - 根据研究目标选择框架
   - 选项包括主题、时间顺序、方法论
   - 考虑复杂主题的混合方法

4. **关键问题制定**:
   - 制定3-5个核心问题来指导综述
   - 确保问题具体但全面
   - 包括事实性和分析性问题

### 性能指标

| 指标 | 描述 | 目标 |
|------|------|------|
| 覆盖广度 | 主题领域的全面性 | 包含所有重要子领域 |
| 来源质量 | 来源的可信度和相关性 | 高质量、代表性来源 |
| 综合深度 | 信息整合为连贯整体 | 清晰识别模式和关系 |
| 差距识别 | 识别知识局限性 | 明确映射未解答问题 |

## 2. 分析协议

**何时使用此协议:**
需要从复杂信息或数据中提取有意义的洞察?此协议引导您进行系统性分析过程——非常适合趋势分析、比较评估、模式识别或批判性评估。

```
提示词: 我需要分析不同全球市场快速采用可再生能源的关键驱动因素。我有关于政策框架、技术成本、市场结构和多个地区投资趋势的数据。我想了解哪些因素最具影响力、它们如何相互作用以及它们在不同市场之间如何变化,以为我们的清洁能源投资组合制定战略洞察。

协议:
/research.analyze{
    intent="通过系统性分析过程提取有意义的洞察",
    input={
        analysis_subject="全球市场可再生能源采用的驱动因素",
        analytical_framework={
            dimensions: ["政策框架", "技术成本", "市场结构", "投资趋势"],
            regions: ["北美", "欧洲", "亚太地区", "新兴市场"],
            timeframe: "过去5年(2020-2025)"
        },
        key_questions=[
            "哪些因素与快速可再生能源采用的相关性最强?",
            "这些因素如何相互作用和影响?",
            "影响因素在不同市场环境中如何变化?",
            "采用率最高的市场出现了哪些模式?"
        ],
        analysis_approach="多维度比较分析与因果关系映射"
    },
    process=[
        /decompose{
            action="将主题分解为可分析的组成部分",
            elements=[
                "识别构成因素和变量",
                "建立相关指标和指示器",
                "映射关系和依赖性",
                "定义分析边界和局限性"
            ]
        },
        /examine{
            action="系统性分析每个组成部分",
            for_each="dimension",
            approaches=[
                "跨地区比较分析",
                "随时间推移的趋势分析",
                "模式识别",
                "相关性和可能的因果关系映射"
            ]
        },
        /contextualize{
            action="考虑相关背景和影响",
            elements=[
                "历史发展和先例",
                "外部因素和影响",
                "系统性约束和促进因素",
                "竞争性或替代性观点"
            ]
        },
        /integrate{
            action="综合组成部分分析为整体理解",
            techniques=[
                "识别跨组件模式",
                "映射因果或影响网络",
                "发展解释性框架",
                "测试替代性解释"
            ]
        },
        /validate{
            action="批判性评估分析结论",
            approaches=[
                "对照可用证据检查",
                "识别假设和局限性",
                "考虑反对论据或例外",
                "评估发现的置信水平"
            ]
        }
    ],
    output={
        key_insights="主要分析发现及支持证据",
        factor_analysis="每个关键因素及其影响的详细检查",
        relationship_map="因素如何互动的可视化或结构化表示",
        strategic_implications="分析发现的实际应用"
    }
}
```

### 实施指南

1. **主题定义**:
   - 清楚阐述分析焦点
   - 定义范围和边界
   - 识别需要检查的具体方面

2. **分析框架选择**:
   - 选择适当的分析维度
   - 定义比较的类别或地区
   - 建立相关时间框架

3. **关键问题制定**:
   - 制定指导分析的具体问题
   - 包括描述性、比较性和因果性问题
   - 关注具有战略或实际价值的问题

4. **分析方法选择**:
   - 选择适合主题和问题的方法论
   - 考虑比较、时间、因果或系统方法
   - 定义适当的分析粒度级别

### 性能指标

| 指标 | 描述 | 目标 |
|------|------|------|
| 分析严谨性 | 系统性、基于证据的检查 | 清晰的逻辑进展 |
| 洞察价值 | 发现的意义和有用性 | 非显而易见、可操作的洞察 |
| 关系映射 | 因素互动的清晰度 | 明确的因果或影响路径 |
| 验证质量 | 结论的批判性测试 | 多种验证方法 |

## 3. 战略前瞻协议

**何时使用此协议:**
需要预测未来发展并为新兴机遇或挑战做准备?此协议引导您进行系统性的未来探索——非常适合趋势分析、情景开发、战略规划或创新预测。

```
提示词: 我需要为未来十年的医疗技术行业制定战略前瞻分析。我们组织需要了解新兴技术、不断变化的患者和提供者需求、潜在的监管变化,以及这些因素可能如何重塑医疗服务模式。我们希望识别战略机遇和潜在的颠覆风险,以指导我们的长期研发投资。

协议:
/research.foresight{
    intent="系统性探索未来发展和战略影响",
    input={
        domain="医疗技术行业",
        time_horizon="10年(2025-2035)",
        focal_areas=[
            "新兴技术及其采用轨迹",
            "演化中的患者和提供者需求与期望",
            "监管环境和政策发展",
            "医疗服务模式转型"
        ],
        key_uncertainties=[
            "AI整合到临床决策的速度和方向",
            "患者数据所有权和隐私监管演化",
            "医疗支付模式转型",
            "公共与私营部门在医疗创新中的角色"
        ],
        strategic_context="为医疗技术公司的长期研发投资决策提供信息"
    },
    process=[
        /scan{
            action="识别和评估变化信号",
            elements=[
                "新兴趋势和发展",
                "潜在的颠覆性力量",
                "系统性变化的弱信号",
                "稳定和约束因素"
            ]
        },
        /analyze{
            action="评估趋势的影响和互动",
            approaches=[
                "相互关联因素的系统分析",
                "利益相关者影响评估",
                "采用和扩散模式分析",
                "监管和政策演化映射"
            ]
        },
        /construct{
            action="开发多个合理的未来情景",
            technique="关键不确定性矩阵",
            elements=[
                "情景叙事和演化路径",
                "关键里程碑和指标",
                "利益相关者立场和响应",
                "每个情景中的关键成功因素"
            ]
        },
        /assess{
            action="评估情景的战略影响",
            for_each="scenario",
            elements=[
                "机遇和挑战",
                "所需能力和资源",
                "竞争定位",
                "风险因素和脆弱性"
            ]
        },
        /recommend{
            action="制定战略选项和监控框架",
            elements=[
                "跨多个未来的稳健策略",
                "对冲和创造选择的方法",
                "早期指标监控系统",
                "适应性战略框架"
            ]
        }
    ],
    output={
        trend_analysis="关键趋势和驱动力的评估",
        scenario_portfolio="一组不同的、合理的未来情景",
        strategic_implications="机遇、挑战和战略选项",
        monitoring_framework="早期指标和适应触发器"
    }
}
```

### 实施指南

1. **领域定义**:
   - 明确指定焦点领域和边界
   - 考虑相关的相邻领域
   - 定义适当的粒度级别

2. **时间跨度设置**:
   - 选择与规划需求相关的时间框架
   - 适当时考虑多个时间跨度
   - 与组织规划周期对齐

3. **焦点领域选择**:
   - 识别探索的关键领域
   - 包括内部和外部因素
   - 考虑技术、社会、经济和监管维度

4. **不确定性识别**:
   - 指出高影响的关键不确定性
   - 关注真正不确定的要素,而非趋势
   - 考虑二阶不确定性和互动

### 性能指标

| 指标 | 描述 | 目标 |
|------|------|------|
| 情景合理性 | 逻辑一致性和可信度 | 无魔法思维或矛盾 |
| 情景区分度 | 未来之间的有意义差异 | 清晰、对比的未来 |
| 战略相关性 | 对决策的可操作影响 | 与战略选择的清晰联系 |
| 监控价值 | 早期预警框架的有用性 | 可观察的、领先的指标 |

## 4. 问题调查协议

**何时使用此协议:**
需要了解原因或维度不清楚的复杂问题?此协议引导您进行系统性的问题探索——非常适合根本原因分析、问题框定、问题诊断或挑战映射。

```
提示词: 我需要调查我们公司在过去18个月客户留存率下降的根本原因。尽管采取了几项改进措施,留存率在大多数客户细分市场中继续下降。我想在制定下一个干预策略之前,全面了解问题,包括潜在根本原因、系统性因素以及与其他业务指标的关系。

协议:
/research.problem{
    intent="系统性调查和理解复杂问题",
    input={
        problem_statement="尽管采取了改进措施,过去18个月客户留存率仍在下降",
        problem_context="B2B SaaS公司,企业客户跨多个行业",
        known_elements={
            symptoms: ["大多数细分市场客户流失增加", "产品使用指标下降", "扩展收入减少"],
            attempted_solutions: ["客户成功团队扩张", "用户界面改进", "新功能添加"],
            affected_stakeholders: ["现有客户", "客户管理团队", "产品开发", "高管领导层"]
        },
        investigation_goals=["识别根本原因", "映射系统性因素", "理解过去解决方案失败原因", "制定全面问题框架"]
    },
    process=[
        /define{
            action="明确问题边界和表现形式",
            elements=[
                "精确的问题定义和范围",
                "关键指标和指示器",
                "历史模式和进展",
                "跨背景或细分市场的变化"
            ]
        },
        /explore{
            action="调查潜在的因果因素",
            approaches=[
                "利益相关者视角分析",
                "跨细分市场比较评估",
                "与外部因素的时间相关性",
                "系统关系映射"
            ]
        },
        /hypothesize{
            action="制定潜在解释",
            techniques=[
                "多假设制定",
                "因果链映射",
                "系统动力学建模",
                "贡献因素加权"
            ]
        },
        /test{
            action="对照可用证据评估假设",
            methods=[
                "证据映射到假设",
                "识别确认/否定数据",
                "评估解释力",
                "考虑替代性解释"
            ]
        },
        /synthesize{
            action="发展整合性问题理解",
            elements=[
                "根本原因识别",
                "系统性因素映射",
                "相互关系分析",
                "全面问题框架"
            ]
        }
    ],
    output={
        problem_analysis="问题及其维度的全面评估",
        causal_model="根本原因和贡献因素及其关系的映射",
        evidence_assessment="关键发现的支持证据评估",
        reframed_problem="澄清的问题陈述及系统性背景"
    }
}
```

### 实施指南

1. **问题陈述制定**:
   - 清楚阐述观察到的问题
   - 关注症状而非假设的原因
   - 包括范围和边界

2. **背景描述**:
   - 提供相关的组织和环境因素
   - 注意历史发展和变化
   - 包括利益相关者格局

3. **已知要素文档化**:
   - 列出观察到的症状和表现形式
   - 记录以前的解决方案尝试和结果
   - 识别受影响的利益相关者和影响

4. **调查目标设置**:
   - 定义调查所需的具体结果
   - 包括分析和实际目标
   - 考虑决策所需的信息

### 性能指标

| 指标 | 描述 | 目标 |
|------|------|------|
| 因果深度 | 识别根本原因 | 超越症状到根本驱动因素 |
| 系统性视角 | 考虑更广泛背景 | 映射关系和互动 |
| 证据质量 | 结论的支持 | 关键发现有多个证据来源 |
| 解决方案路径 | 从理解到行动的清晰路径 | 干预的可操作影响 |

## 5. 比较评估协议

**何时使用此协议:**
需要系统性地比较多个选项、方法或实体?此协议引导您进行结构化比较——非常适合解决方案评估、竞争分析、方法评估或决策支持。

```
提示词: 我需要对三种领先的企业数据库技术(Oracle、Microsoft SQL Server和PostgreSQL)进行全面比较,以确定最适合我们组织新数据平台的技术。我们需要评估性能特征、成本结构、可扩展性、安全功能、生态系统支持和未来发展路线图,以做出明智的技术选择决策。

协议:
/research.compare{
    intent="使用结构化框架系统性比较多个选项或实体",
    input={
        comparison_subjects=["Oracle数据库", "Microsoft SQL Server", "PostgreSQL"],
        evaluation_dimensions=[
```

## 平衡严谨性和创造性

研究协议提供结构而不扼杀发现。考虑这些平衡原则:

1. **方法与开放性**:建立方法论严谨性,同时保持对意外发现的开放
2. **结构与探索**:创建包含发散式调查的结构化过程
3. **精确性与适应性**:开发能够适应新兴洞察的精确方法
4. **效率与彻底性**:构建保持全面覆盖的高效工作流程

成功的研究协议创建了确保质量同时支持发现的框架。

## 结论:知识发现的演化

研究协议将通常混乱的调查过程转化为结构化、可靠的洞察路径,而不牺牲发现和创造性的基本要素。通过为研究过程提供明确的架构,它们使更系统化、高效和高质量的知识开发成为可能。

在构建研究协议库时,请记住这些原则:

1. **从关键问题开始**:关注最能从结构中受益的研究挑战
2. **平衡严谨性和发现**:创建足够的方法论结构而不约束探索
3. **基于结果迭代**:根据研究结果优化协议
4. **适应背景**:为特定领域和问题修改协议
5. **内置质量**:在多个阶段整合验证和批判性评估

有了这些原则和本指南中的研究协议,您已经具备了将不可预测的研究过程转化为可靠、严谨的调查的能力,始终如一地产生有价值的洞察。
